**What is a container ?**
A container that packages up code and all its dependencies so the application runs quickly and reliably from one computing environment to another. 
A Docker container image is a lightweight, standalone, executable package of software which includes everything needed to run an application: code, runtime, system tools, system libraries and settings.

Container is a bundle of Application, Application libraries required to run your application and the minimum system dependencies.

Containers are lightweight which allows containers to share the host operating system's kernel and libraries, while still providing isolation for the application and its dependencies. This results in a smaller footprint compared to traditional virtual machines, as the containers do not need to include a full operating system. 
Additionally, Docker containers are designed to be minimal, only including what is necessary for the application to run, further reducing their size.
