**What is Docker and Container ?**

"Docker" is the base-layer where you set/run your containers, "Docker image- *ubuntu,linux" is required to set the environment that Containers can run. 
Docker technology is standalone, and lightweight executable package of software to host and run your Containers.

Well, "Container" are packaged up programming code, bundle of applications, application libraries which use it's Host **Docker** operating system's kernel and libraries, while providing isolation for its own application code's and its dependencies.

** Remember: Container can't run itself without Docker image!!

**Docker Containers vs Virtual Machine** 

Containers uses it's HOST- Docker's operating system kernel, libraries which make Container's to work fast and executing it's own programing code.  
Virtual Machine(VMs) require a full-fledged Operating System and compatible Hypervisor to run, which is highly resource-intensive (compute,storage,RAM).
